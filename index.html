The existing fuzzing research environment was pretty broken — I rebuilt it from scratch using Google's FuzzBench, Docker, and a custom LLVM compiler build. Setup time dropped by ~60% and experiments became fully reproducible.

I also implemented two new statistical models from a research paper to better predict when testing should stop, improving accuracy by 30%+. Optimized the Docker layer caching to cut rebuild time by ~70%.

MetARAG — AI Chatbot for Document Search & Retrieval

Led a team of 5+ to build an AI system that lets you actually talk to large collections of PDFs — ask questions, get answers with context, find things instantly. Built with Python, LangChain, and OpenAI APIs.

I designed the whole data pipeline and optimized it with GPU acceleration — processing efficiency improved by ~65% and responses came back 40% faster across 4+ different document sets.

GitHub
ECI Pipeline - Ecosystem Change Intelligence for Android Risk

Built an intelligence pipeline that monitors 10 Android security, CVE, OEM bulletin, developer-doc, and policy sources, detects what changed, and turns those deltas into evidence-backed Action Tickets for fraud and risk operations teams.

The system combines DeltaRAG, Graph-RAG, pgvector, NetworkX, Groq LLM agents, and a Next.js dashboard. A Sentinel Agent scores changes for relevance and risk, then a Coordinator Agent uses cross-source evidence to recommend concrete actions.

GitHub
Movie Recommendation System — Full ML Pipeline from Model to Production

Built and deployed a movie recommendation service end-to-end with a 6-person team — model training, REST API (FastAPI + Docker), CI/CD with Jenkins, and live monitoring with Prometheus and Grafana. Hit RMSE 0.58 and 65% faster inference.

We also cared about doing it right — built A/B testing, drift detection, fairness checks, and security mechanisms (rate limiting, poisoning detection) into the system from the start.

GitHub
End-to-End Encrypted Messaging App — MTProto 2.0

Built a secure messaging application by implementing Telegram's MTProto 2.0 protocol from scratch — including AES-256 encryption, SHA-256 hashing, and Perfect Forward Secrecy so no two sessions share keys.

Supports multiple transport layers (TCP, WebSocket, HTTP/HTTPS) with full message serialization, session handling, and time sync. Basically rebuilt how Telegram's security layer works at the protocol level.

GitHub
